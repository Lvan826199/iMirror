# 已知问题与归因（防复发清单）

真机联调阶段暴露的 bug 全部记录在此，按**根因**归类。改代码/加功能前先扫一遍
对应类别的"防复发规约"，避免重蹈覆辙。每条附修复 commit 便于回溯。

---

## A 类：pyusb 异常类型捕获不全 ⭐ 最高频

**现象**：doctor / gui / 打开设备时直接抛 Traceback 崩溃，而非给出友好提示。

**已发生的具体 bug**：
| 现象 | 抛出的异常 | 修复 commit |
| --- | --- | --- |
| Windows 未换驱动，doctor 读序列号崩溃 | `ValueError: has no langid` | eb5d5de |
| gui 缺 av/cv2 依赖时崩栈 | `ImportError`（且 import 在函数体内，except 位置错） | e8b584a |
| Windows 切 USB 配置崩溃 | `NotImplementedError: Operation not supported` | 6a56b3a |

**根因**：pyusb 底层（libusb1 backend）**不只抛 `usb.core.USBError`**。读字符串描述符
失败抛 `ValueError`，后端不支持的操作抛 `NotImplementedError`，还有 `NotImplementedError`
被包在 `_check()` 里。只捕获 `USBError` 必然漏网。

**防复发规约**：
- 任何 pyusb 调用点（`get_string` / `set_configuration` / `ctrl_transfer` /
  `read` / `write` / `get_active_configuration`），捕获异常一律写全三元组：
  `except (usb.core.USBError, NotImplementedError, ValueError)`。
- 延迟 import 的可选依赖（av/cv2），try 要**把使用点也包进去**，不能只包 import 行。

## B 类：USB 字符串描述符含不可见字符

**现象**：`--udid 00008110000275943EEB801E`（从 doctor 输出复制的）报"未找到设备"，
但 doctor/devices 又明明列出了这台。

**根因**：iPhone 的 USB 序列号/产品名描述符带 **NUL 填充**（`\x00`），打印时不可见，
但字符串**比较时不相等**。

**防复发规约**：
- 所有从 USB 描述符读来的字符串，一律过 `discovery.clean_str()`（去 NUL + strip）
  再使用/比较。新增读描述符的地方记得套。
- 面向用户的匹配（如 `--udid`）再做一层归一化：去横杠、转小写（见 F 类）。

修复 commit：8574a22

## C 类：Windows USB 平台特性（非 bug，是必须适配的现实）

Windows 的 USB 栈与 Linux/macOS 差异大，以下都不是代码错误，而是**必须处理的平台现实**：

1. **复合设备的"复合父设备"默认被 Zadig 隐藏**
   - iPhone 是 USB 复合设备，Zadig 的 `Ignore Hubs or Composite Parents` 默认勾选，
     会藏掉需要换驱动的父节点。只给子接口换驱动**不生效**。
   - 处理：手册明确要求取消勾选该项，按 USB ID `05AC 12A8` 选 Composite Parent。

2. **一台设备有"两个形态"，各记各的驱动**
   - iPhone 普通形态 vs 激活投屏后的新形态，Windows 视为两个设备实例。
     换过普通形态的驱动后，激活切到新形态，Windows 又派默认驱动 → 需**二次换驱动**。
   - 处理：激活失败的报错内嵌自救指引（重开 Zadig 给新形态换 libusbK）。

3. **激活后重枚举慢**
   - Windows 首次给新形态装驱动可能 >5 秒，原轮询 5 秒（10×0.5s）超时。
   - 处理：轮询加长到 15 秒（30×0.5s）。修复 commit：8574a22

4. **`libusb_set_configuration` 返回 NOT_SUPPORTED**
   - WinUSB/libusbK 驱动模型下，USB 配置由 OS/驱动在系统层选定，**libusb 无法切换
     活动配置**（这是 libusb 在 Windows 的固有限制，非本项目 bug）。
   - 关键认知：设备激活后，QT 配置**本就是**当前活动配置（discovery 能找到 0x2A
     接口就证明它在活动配置里），所以根本**不需要**切配置。
   - 处理：`adapter.open` 逐级降级——活动配置已是目标则跳过切换 →
     `set_configuration` → 裸 `SET_CONFIGURATION` 控制请求 → 全失败也继续在当前
     配置里找 QT 接口。修复 commit：6a56b3a

**防复发规约**：
- 新增任何 USB 操作前，先想"这在 Windows 的 WinUSB/libusbK 下支持吗"，
  默认给降级路径，不要假设 Linux 的行为在 Windows 成立。
- Windows 特有的操作现实同步进 `docs/真机联调手册.md` 的排障小节。

## D 类：判定逻辑对复合设备过严

**现象**：换好驱动后 doctor 仍报失败——它要求"所有 Apple USB 节点都可访问"，
但复合设备天然有一批读不了的子接口节点。

**根因**：把"部分节点不可访问"当成了"环境不可用"，一票否决。

**防复发规约**：
- 判定"环境是否就绪"以**能否识别出目标 iOS 设备**为准，不以"所有节点都完美"为准。
  不可访问的节点降级为 ⚠ 提示，不阻断。

修复 commit：7da25fb

## E 类：脚本 / 工具链

1. **bat 在 UTF-8 代码页（chcp 65001）下，含中文+英文括号的 echo 行解析出错**
   - 报 `'xxx)复查' is not recognized as an internal or external command`。
   - 处理：结尾提示文案**去掉所有英文括号**（转义 `^(` 也不完全可靠）。修复：35e8f7e
2. **一键脚本依赖装不全**
   - 初版只装 `.[dev,windows]`，跑 gui 时缺 av/cv2。
   - 处理：脚本统一装 `.[dev,windows,gui]` / `.[dev,gui]`，用户无需手动补装。修复：5e3a3f8
3. **`uv sync` 会拆掉环境**
   - 它按 pyproject 精确同步，把 dev/windows/gui 附加项全卸载（含 libusb-package）。
   - 处理：手册警告不要用 `uv sync`，装包统一 `uv pip install`；误用后重跑一键脚本恢复。

**防复发规约**：
- 改 bat 提示文案：不用英文括号，改用中文顿号/破折号或分行。
- 改依赖分组（pyproject optional-dependencies）时，同步改两个安装脚本的 extras。

## F 类：用户输入容错

**现象**：`--udid` 传入 usbmuxd 风格带横杠的 `00008110-000275943EEB801E` 报未找到。

**根因**：USB 序列号不带横杠，usbmuxd/Xcode 显示的 udid 带横杠，两者格式不一。

**防复发规约**：
- 面向用户的标识匹配做归一化（去横杠 + 转小写）后比较，两种写法都接受。
- 匹配失败时，报错里**列出当前可见的合法值**，让用户直接照抄。

修复 commit：6a56b3a

## 通用流程规约（本项目长期要求）

1. **改代码必同步文档**：动了 CLI 参数/行为、依赖分组、协议逻辑，
   同一次提交里改掉 README / 手册 / 协议速查 / AGENTS.md 对应处。
2. **每个 bug 必归因**：修完 bug 在本文件对应类别追加一行（现象+根因+commit），
   新根因就新开一类。目的是让"同类错误"一次绝根。
