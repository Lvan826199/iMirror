# 真机联调手册

Windows / macOS / Linux 都可以做联调，一步一步照做即可。
每一步都写了**预期结果**和**不对时看什么**。遇到卡住的地方，按最后一节收集信息。

## 0. 你需要什么

- 一台 iPhone 或 iPad（不需要越狱，系统版本不限，越新越有验证价值）
- 一根**支持数据传输**的数据线（纯充电线不行，这是最常见的坑）
- Windows / macOS / Linux 电脑任意一台

## 1. 环境准备（按你的系统看对应小节，其余跳过）

> Windows 和 macOS 都有**一键安装脚本**（在 `scripts/` 目录），
> 自动完成：装 uv → 建虚拟环境 → 装依赖 → 跑离线测试 → 环境自检。
> 脚本失败时再按各小节的手动步骤逐条做。

### Windows

**一键安装**：克隆/下载本仓库后，双击运行

```
scripts\setup-windows.bat
```

（会自动装 uv、依赖、ffmpeg 并跑自检；窗口最后会提示驱动替换步骤。）

**脚本无法代劳、必须手动做一次的——换驱动**：
1. `Win+R` 输入 `services.msc`，找到 **Apple Mobile Device Service**，
   停止并把启动类型设为"禁用"（没有该服务则跳过，说明没装过 iTunes）
2. 下载 [Zadig](https://zadig.akeo.ie/)（免安装的驱动替换工具）
3. 打开 Zadig：Options → 勾选 **List All Devices** → 下拉框选中 iPhone →
   目标驱动选 **libusbK** → **Replace Driver**
4. 想恢复 iTunes 同步时，在设备管理器里把 iPhone 驱动改回来即可，随时可逆

<details><summary>手动安装步骤（脚本失败时展开）</summary>

```powershell
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
uv venv .venv
.venv\Scripts\Activate.ps1
uv pip install -e ".[dev,windows]"     # windows 附加项自带 libusb, 免手动放 dll
winget install --id Gyan.FFmpeg -e     # 可选, 播放验证用
```

</details>

### macOS

**一键安装**：终端进入项目目录，执行

```bash
bash scripts/setup-macos.sh
```

（会自动 brew 装 libusb/uv/ffmpeg、建环境、装依赖并跑自检；
未装 Homebrew 时脚本会给出安装命令。）

- **不需要 sudo**，macOS 允许用户进程访问 USB 设备
- 联调时先退出 QuickTime、爱思助手等可能占用 iPhone 的程序
- ⚠️ macOS 路线尚未真机验证过（macOS 本身自带投屏，本项目在 mac 上主要是开发调试用）

<details><summary>手动安装步骤（脚本失败时展开）</summary>

```bash
brew install libusb uv ffmpeg
uv venv .venv
source .venv/bin/activate
uv pip install -e ".[dev]"
```

</details>

### Linux

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh    # 没装 uv 先装
uv venv .venv
source .venv/bin/activate
uv pip install -e ".[dev]"
```

USB 权限二选一：

**方式 A：一次性配置 udev 规则（推荐）**

```bash
sudo tee /etc/udev/rules.d/39-imirror.rules <<'EOF'
SUBSYSTEM=="usb", ATTR{idVendor}=="05ac", MODE="0666"
EOF
sudo udevadm control --reload-rules && sudo udevadm trigger
```

配置后重新插拔手机，之后所有命令都**不需要 sudo**。

**方式 B：临时用 sudo（快速验证用）**

没配 udev 规则时，把下文所有 `python -m imirror ...` 命令改成
`sudo .venv/bin/python -m imirror ...`。

> 下文命令统一写成 `python -m imirror ...`，前提是已激活 venv
> （Windows: `.venv\Scripts\Activate.ps1`；macOS/Linux: `source .venv/bin/activate`）。
> **sudo 只有 Linux 且没配 udev 规则时才需要，Windows/macOS 不用。**

## 2. 第 0 步：环境自检（强烈建议先跑）

```bash
python -m imirror doctor
```

它会逐项检查 pyusb → libusb → 设备发现 → 访问权限 → QT 配置状态，
**每一项不通过都会给出你当前系统对应的修复命令**。全部 ✓ 再往下走。

```
平台    : Windows (...)
Python  : 3.12.x
✓ pyusb 已安装
✓ libusb 后端可用
✓ 发现 1 个 Apple USB 设备
✓ 设备可访问 (权限/驱动正常)
✓ 00008110-xxxx  iPhone  QT配置: 未激活 (record 时会自动激活)

环境就绪, 下一步: python -m imirror record out.h264 out.wav --duration 10
```

## 3. 分步联调

### 第 1 步：连接并信任

用数据线连接手机 → 解锁手机 → 弹出「信任此电脑？」→ 点信任。

> Windows 上信任弹窗可能需要 Apple Mobile Device Service 在运行；
> 如果换驱动后不弹了，先恢复驱动/服务→完成信任→再换回 libusbK。
> Linux 上信任弹窗依赖 usbmuxd 服务（一般随系统自带）。

### 第 2 步：确认能看到设备

```bash
python -m imirror devices
```

**预期输出**（一行一台设备）：

```
00008110-000A1B2C3D4E5F  iPhone  vid:pid=05ac:12a8  QT配置: 未激活
```

**不对时**：回去跑 `python -m imirror doctor`，按它的提示修。

### 第 3 步：激活 QuickTime 配置 ⚠️ 联调重点 1

```bash
python -m imirror -v activate
```

**预期**：手机会「断开又出现」一次（系统可能有提示音），几秒内打印：

```
xxx QT 配置已激活 (config #N)
```

再跑一次 `devices`，QT配置 应显示「已激活」。

**这一步验证的是代码里的第一个 TODO**：激活请求发出后设备重新枚举，
`activation.py` 轮询 10 次 × 0.5 秒等它回来。**不对时**：
- 报「无法激活」→ 重枚举比 5 秒慢。把 `-v` 日志发出来，
  并告知从手机断开到重新出现大概几秒（Linux 看 `dmesg -w`，Windows 听设备插拔音）
- Windows 激活后设备"消失" → 重枚举后的新配置可能也需要在 Zadig 里再换一次驱动

### 第 4 步：录制 10 秒 ⚠️ 联调重点 2（最关键的一步）

```bash
python -m imirror -v record out.h264 out.wav --duration 10
```

**预期 `-v` 日志按这个顺序出现**（顺序就是协议握手）：

```
已 claim QuickTime 接口 #N (in:0x8x out:0x0x)
收到 PING, 回复 PING
发送 ASYN HPD1 x2
回复 CWPA clockRef=xxxx
发送 ASYN HPA1
发送首个 NEED, clockRef=xxxx
音频格式: AudioStreamBasicDescription{...48000...}
写入参数集: FormatDescriptor{video 886x1920, codec:avc1, sps:xxB, pps:xxB}   ← 最重要的一行
  [    5s] 视频 xxx 帧 (~30.0fps, x.xMB)  音频 xxx 帧 (xxxKB)
  [   10s] ...
请求设备停止推流...
会话已关闭 (视频帧:N 音频帧:M)
```

**核对点**：
1. `写入参数集` 这行必须出现，且 sps/pps 字节数 > 0
   —— 这验证 `formatdescriptor.py` 的 SPS/PPS 提取位置（代码里标了 TODO 的地方）
2. 统计行的视频帧数持续增长（说明 FEED→NEED 流控正常）
3. 结束时「会话已关闭」前**没有** RELS 超时告警
   —— 这验证按 Go 版修正过的 `close_session` 关闭时序

**不对时**：
- 卡在 claim 接口 / 报 `Resource busy` → Linux: `sudo systemctl stop usbmuxd`；
  Windows: 确认 Apple Mobile Device Service 已停用；macOS: 退出 QuickTime 等
- 收到 PING 后没有下文 → 把完整日志发出来（可能是分帧或首包问题）
- 没有「写入参数集」→ **这就是 TODO 命中的情况**，执行
  `python -m imirror -v record out.h264 out.wav --duration 10 2>&1 | tee record.log`
  （PowerShell 用 `... | Tee-Object record.log`）保留日志，
  这是修 `formatdescriptor.py` 需要的第一手数据
- 帧数不增长/几秒后停 → FEED/NEED 流控问题，同样保留完整日志

### 第 5 步：验证录制产物

```bash
ffplay -f h264 out.h264        # 应该能看到手机屏幕画面
ffprobe -f h264 out.h264       # 应显示正确分辨率, h264 (High) 之类
ffplay out.wav                 # 应该有手机的声音(录制时放段音乐)
```

（没装 ffmpeg：Windows `winget install ffmpeg`；macOS `brew install ffmpeg`；
Ubuntu `sudo apt install ffmpeg`。也可以用 VLC 直接打开这两个文件。）

**核对点**：画面不花屏、分辨率与手机匹配、音频时长 ≈ 10 秒。

- 画面花屏/绿屏但能播 → SPS/PPS 内容可能不对，保留 out.h264 开头数据：
  Linux/macOS `xxd out.h264 | head -60 > h264head.txt`；
  Windows `certutil -encodehex out.h264 h264head.txt` 后取开头 60 行
- wav 是静音 → 录的时候手机放点声音再试一次；仍静音则保留 wav 和日志

### 第 6 步：实时预览（可选）

```bash
uv pip install -e ".[gui]"     # 首次需要(安装 av + opencv)
python -m imirror gui          # 按 q 退出
```

**预期**：弹出窗口显示手机画面，延迟 < 1 秒。卡顿属于已知待优化项（README Roadmap）。

### 第 7 步：恢复手机正常模式

正常退出（Ctrl+C 或 --duration 到时）后设备会自动释放；如果电脑不识别手机了，
重新插拔一次数据线即可恢复。Windows 想恢复 iTunes 同步需在设备管理器把驱动换回来。

## 4. 联调验收清单

全部打勾即可宣布真机联调完成，把结果（含操作系统与 iOS 版本）同步到 AGENTS.md 的「当前状态」：

- [ ] `doctor` 全部 ✓
- [ ] `devices` 能列出设备
- [ ] `activate` 激活成功（重枚举 ≤ 5 秒）
- [ ] `record` 完整跑通 10 秒无报错
- [ ] 日志出现「写入参数集」且 sps/pps > 0 字节（formatdescriptor.py TODO 销案）
- [ ] `ffplay` 能正常播放 out.h264，画面正确
- [ ] out.wav 有声音
- [ ] 结束时无 RELS 超时告警（close_session 时序验证）
- [ ] 连续录 3 分钟以上不断流（SKEW 应答验证）

## 5. 出问题时收集什么

把以下内容打包，能极大加快远程定位：

```bash
# 1. 环境自检结果
python -m imirror doctor > doctor.txt 2>&1
# 2. 完整 DEBUG 日志
python -m imirror -v record out.h264 out.wav --duration 10 > record.log 2>&1
# 3. 系统侧 USB 信息
#    Linux:   lsusb -d 05ac: -v > usb.txt 2>/dev/null; dmesg | tail -100 > dmesg.txt
#    Windows: 设备管理器截图(iPhone 所在节点 + 当前驱动名)
#    macOS:   system_profiler SPUSBDataType > usb.txt
```

外加一句话描述：卡在第几步、现象是什么、手机型号和 iOS 版本。
