# 真机联调手册

当前 iMirror 只验收 **macOS 原生投屏路线**。Windows 暂时不做兼容，Linux 暂未完成真机验收。

## 1. 准备

你需要：

- 一台 Mac
- 一台 iPhone 或 iPad
- 一根支持数据传输的数据线
- Xcode Command Line Tools
- Homebrew 和 uv（脚本会尽量自动安装）

安装环境：

```bash
cd ~/imirror
bash scripts/setup-macos.sh
```

手动安装：

```bash
uv venv .venv
uv pip install --python .venv/bin/python -e ".[dev,gui]"
```

## 2. 连接设备

1. 退出 QuickTime、爱思助手、录屏工具等可能占用 iPhone 投屏的程序。
2. 用数据线连接 iPhone/iPad 和 Mac。
3. 解锁设备。
4. 设备弹出“信任此电脑”时点击“信任”，并输入锁屏密码。

## 3. macOS 原生投屏验证

列出屏幕源：

```bash
.venv/bin/python -m imirror macos-devices
```

预期能看到类似：

```text
00008110-000275943EEB801E  iPhone  model=iOS Device  type=AVCaptureDeviceTypeExternal
```

录制 10 秒：

```bash
.venv/bin/python -m imirror macos-record native.mov --duration 10
```

打开预览窗口：

```bash
.venv/bin/python -m imirror macos-gui
```

首次运行会编译并 ad-hoc 签名 Swift helper，缓存到：

```text
~/Library/Caches/imirror/
```

后续会复用缓存 helper，减少 TCC 权限和临时脚本身份带来的干扰。

## 4. 屏幕录制权限

如果提示 Screen Recording permission denied：

1. 打开 macOS `系统设置 -> 隐私与安全性 -> 屏幕录制`。
2. 给当前终端、IDE 或启动 iMirror 的 App 授权。
3. 完全退出并重新打开终端/IDE。
4. 重新运行 `macos-devices` / `macos-record` / `macos-gui`。

## 5. 常见问题

### `macos-devices` 看不到设备

- 确认设备已解锁并信任电脑。
- 换一根支持数据的数据线。
- 退出 QuickTime、会议软件、录屏工具。
- 重插设备后再试。

### `recording failed before start` 或 `valeria connection not seen`

这通常表示 macOS 的 `iOSScreenCaptureAssistant` 没能让设备进入 Valeria 出流状态。
先不要归因到 Python 协议代码。

按顺序尝试：

```bash
.venv/bin/python -m imirror reset
```

然后：

1. 重插数据线。
2. 保持设备解锁。
3. 退出 QuickTime 等占用程序。
4. 用 QuickTime Player 自己选择 iPhone 做交叉验证。
5. 仍失败时重启 iPhone。
6. 仍失败时重启 Mac。

如果 QuickTime Player 也提示“这项操作无法完成”，并且系统日志里有
`StartStream throwing valeria connection not seen`，说明系统原生投屏链路本身没有建立，
iMirror 暂时无法绕过。

### 某台设备失败，但其他设备正常

优先按设备/macOS 临时 Valeria 状态卡死处理：`reset`、重插线、重启设备。
此前实测同一台 Mac 上多台 iOS 设备可正常 QuickTime 投屏，单台设备重启后也可恢复。

## 6. raw USB 研究命令

以下命令保留给协议研究和排障，不作为当前用户交付路径：

```bash
.venv/bin/python -m imirror doctor
.venv/bin/python -m imirror devices
.venv/bin/python -m imirror activate
.venv/bin/python -m imirror record out.h264 out.wav --duration 10
.venv/bin/python -m imirror reset
```

raw USB 路线如果 0 帧、bulk 超时或只有半激活状态，请优先使用 macOS 原生后端验证。

## 7. 问题收集

反馈问题时请收集：

```bash
.venv/bin/python -m imirror macos-devices > macos-devices.txt 2>&1
.venv/bin/python -m imirror macos-record native.mov --duration 10 > macos-record.log 2>&1
.venv/bin/python -m imirror reset > reset.txt 2>&1
system_profiler SPUSBDataType > usb.txt
```

同时说明：

- Mac 型号和 macOS 版本
- iPhone/iPad 型号和 iOS 版本
- 是否已给屏幕录制权限
- QuickTime Player 是否能选择该设备投屏
